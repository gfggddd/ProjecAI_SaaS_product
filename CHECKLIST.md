## 🚀 Как дублировать проект для нового бизнеса

### Способ 1: Автоматический (рекомендуется)

```bash
# Создайте новый проект для вашего бизнеса
python duplicate_for_business.py --name "My Salon" --type beauty --folder ~/my-salon-bot

# Перейдите в папку
cd ~/my-salon-bot

# Инициализируйте
python init_project.py

# Отредактируйте конфиг
nano .env

# Запустите
python main_tg.py
```

### Способ 2: Ручной

```bash
# 1. Скопируйте всю папку Project.AI
cp -r Project.AI my-salon-bot
cd my-salon-bot

# 2. Удалите .env с реальными данными
rm .env

# 3. Скопируйте .env.example
cp .env.example .env

# 4. Отредактируйте для вашего бизнеса
nano .env
nano BASE_VECTOR/barber_data.txt

# 5. Инициализируйте
python init_project.py

# 6. Запустите
python main_tg.py
```

## 🔄 Компоненты системы

### 1. Telegram бот (aiogram 3.x)
- **Поддерживает:** CREATE, UPDATE, CANCEL действия
- **Оптимизация:** asyncio.create_task() для фоновых операций
- **Логирование:** Через logger_config.py

### 2. WhatsApp бот (FastAPI webhook)
- **API:** Meta Cloud API v17+
- **Оптимизация:** Асинхронные операции в фоне
- **Масштабирование:** Готов к высоким нагрузкам

### 3. ИИ-агент (DeepSeek)
- **Модель:** deepseek-chat
- **RAG:** Chroma DB для поиска информации о бизнесе
- **Экстракция:** JSON структура данных из диалога

### 4. База данных
- **По умолчанию:** Google Sheets (легко адаптируется)
- **Можно заменить:** PostgreSQL, MongoDB, SQLite
- **Функции:** Создание, обновление, отмена записей

### 5. Логирование
- **Файлы:** logs/app.log
- **Консоль:** Вывод в терминал
- **Уровни:** DEBUG, INFO, WARNING, ERROR, CRITICAL

## 📈 Масштабирование

### Для 10-100 запросов/день
- Текущая архитектура (Google Sheets + один сервер)
- Все работает отлично из коробки

### Для 100-1000 запросов/день
- Замените Google Sheets на PostgreSQL (edit database/postgres.py)
- Добавьте Redis для кэширования
- Используйте docker-compose.yml для оркестрации

### Для 1000+ запросов/день
- Несколько инстансов ботов за Load Balancer
- Очередь задач (Celery, RQ)
- CDN для статических ресурсов
- Мониторинг (Prometheus, Grafana)
